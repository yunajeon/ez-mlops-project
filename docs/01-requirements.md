# 요구사항 분석

## 전제/제약
- 팀 구성: 개발자 5명, 전담 운영자 없음 → **수동 운영 금지(완전 자동화)**
- 모델 업데이트: 월 1~2회
- 가용성: 99.5% 이상
- OS: Ubuntu 22.04
- 외부 SaaS(모니터링/로깅/APM) 사용 불가 → OSS self-hosted

## 기능 요구사항
- 감성 분석 REST API 제공
- 헬스체크/레디니스 제공
- 모델 버전/서비스 버전 확인 엔드포인트
- 자동화된 테스트/배포 파이프라인

## 비기능 요구사항
- 장애 격리(프로브/재시작/롤백)
- 관측성(메트릭/로그/트레이스)
- 보안(최소권한/비밀관리/공급망)

## 성공 기준
- `docker compose up` 한 번으로 시스템 재현 가능
- 모델 업데이트 시 자동 배포 및 안전 롤백 가능(설계+구현)
