# 요약 (1페이지)

## 핵심 목표
- 월 1~2회 업데이트되는 감성 분석 모델을 **운영자 없이** 안전하게 지속 배포/운영
- 가용성 99.5% 이상(무중단 롤링/헬스체크/관측성/롤백)

## 핵심 전략 3가지
1) 모델을 이미지에 포함한 배포(버전=이미지 태그)로 재현성/롤백성 확보  
2) GitOps 기반 CD(예: Argo CD)로 선언적 운영 + 완전 자동화  
3) OSS Observability(메트릭/로그/트레이스)로 장애 탐지/원인분석 자동화

## 트레이드오프
- 이미지 크기 증가 ↔ 런타임 다운로드 실패 제거/롤백 단순화
- 카나리(복잡) ↔ 99.5% 목표에 필요한 최소 안정성(롤링+프로브+알림) 우선
- 고성능 최적화(ONNX/양자화)는 선택사항으로 문서화

## 성능(내 환경 기준)
- (bench 결과 붙여넣기)
- p50/p95/p99, QPS, CPU/RAM 사용량

## 운영 시나리오(모델 업데이트)
- AI 팀이 새 모델 버전 전달 → 레포/태그 업데이트
- CI: 테스트→이미지 빌드(모델 포함)→스캔→푸시→매니페스트 업데이트
- CD: 변경 감지→롤링 배포→SLO 위반 시 자동 롤백(설계)
